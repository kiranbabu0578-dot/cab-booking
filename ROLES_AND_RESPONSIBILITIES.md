# Roles and Responsibilities – Cab Booking System

## Overview

The Cab Booking System consists of three primary roles: **User (Rider)**, **Driver**, and **Admin**. Each role has specific responsibilities that ensure the smooth operation of the application.

---

# User (Rider)

### Purpose

The User (Rider) is the customer who uses the application to book and manage cab rides.

### Responsibilities

* Register a new account.
* Login securely to the application.
* Search for available cabs.
* Book a cab by selecting pickup and drop locations.
* Track the ride in real time.
* View ride history.
* Download or view ride receipts.
* Cancel rides before the driver starts the trip.
* Update personal profile information.

### Real-Life Example

A customer opens the Cab Booking app, logs in, books a cab to their destination, tracks the driver's location, completes the journey, and later checks the ride history and receipt.

---

# Driver

### Purpose

The Driver provides transportation services and manages assigned ride requests.

### Responsibilities

* Login to the driver dashboard.
* View assigned ride requests.
* Accept or reject ride requests.
* Navigate to the customer's pickup location.
* Update ride status (Accepted, Started, Completed).
* View completed rides.
* Check earnings and ride history.
* Update availability status (Online/Offline).

### Real-Life Example

A driver receives a ride request, accepts it, picks up the passenger, completes the journey, and checks the earnings for the completed ride.

---

# Admin

### Purpose

The Admin manages the overall Cab Booking System and ensures efficient operation.

### Responsibilities

* Manage user accounts.
* Manage driver accounts.
* Approve or deactivate drivers.
* Monitor all ride bookings.
* Monitor payments and transactions.
* View reports and analytics.
* Add, update, or remove vehicle categories.
* Resolve customer complaints and system issues.

### Real-Life Example

The administrator monitors daily bookings, reviews payment reports, manages registered users and drivers, and updates vehicle categories whenever necessary.

---

# Role Summary

| Role         | Main Responsibility                                                             |
| ------------ | ------------------------------------------------------------------------------- |
| User (Rider) | Register, login, book cabs, track rides, and view ride history.                 |
| Driver       | Accept ride requests, complete rides, update ride status, and monitor earnings. |
| Admin        | Manage users, drivers, bookings, payments, reports, and vehicle categories.     |

---

# Conclusion

The Roles and Responsibilities document clearly defines the permissions and duties of each user type within the Cab Booking System. Proper role management improves system security, simplifies operations, and ensures a better experience for customers, drivers, and administrators.
